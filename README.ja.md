# Universal Mirror

[HTTrack](https://www.httrack.com/) を駆動する、ターミナル完結の対話型 **TUI ラッパー**。通常サイトも Internet Archive (Wayback Machine) も、macOS / Linux でミラーできる。

> 🇬🇧 English version: [README.md](README.md)

---

## これは何で、何ではないか

これは**新しいダウンロードエンジンではない**。`httrack` を呼び出しているだけで、クロール・ダウンロード・リンク書き換えはすべて HTTrack 本体がやっている。このスクリプトが足しているのは、その上に乗る**ターミナルネイティブな快適操作レイヤー**:

- `raspi-config` 風の対話 TUI（`whiptail` 利用）。HTTrack の長いオプション文字列を覚える必要がない。
- Wayback Machine URL の自動判定と、手作業だと面倒な日付パターン処理。
- 編集・拡張しやすい、整理済みのデフォルト除外リスト（SNS・トラッカー・アナリティクス・検索エンジン）。
- 妥当な安全装置（削除確認、出力パスのガード、数値検証、全画面でのキャンセル処理）。

GUI アプリ（WinHTTrack, SiteSucker）と素の CLI（`wget` / `httrack`）の隙間を埋める。つまり **ターミナル内で完結する、ガイド付きの操作体験**。SSH 越し、ヘッドレスサーバ、あるいは「素の `wget` のフラグは面倒だが GUI は使いたくない」人に向く。

---

## 制限事項（使う前に読むこと）

HTTrack をラップしている以上、HTTrack の限界をそのまま引き継ぐ:

- **JavaScript 重量級 / 動的サイト（SPA）は完全には取得できない。** HTTrack は静的アセットと HTML をダウンロードするが、実行時にクライアント側でレンダリングされるコンテンツは基本的に取れない。JS レンダリングが必要なら、ブラウザエンジン系ツール（SingleFile、ブラウザベースのアーカイバ）の方が適切。
- **Wayback Machine 対応はベストエフォートで、完璧ではない。** HTTrack は Internet Archive の `/web/<タイムスタンプ>/` リンクプレフィックスを完全には書き換えられず、一部のリソースバリアント（`_if_`, `_cs_`, `_js_` など）を取りこぼす。除外フィルタで緩和はしているが、ローカルで Wayback ミラーを開くと内部リンクが一部切れることは想定しておくこと。高精度な Wayback アーカイブが必要なら、専用ツール（例: `waybackpack`）を検討。
- **通常（ライブ）サイトのミラーが完全サポート対象。** Wayback は利便性レイヤーという位置づけ。

## 責任ある利用

このツールはデフォルトで `--robots=0`（`robots.txt` 無視）を設定し、ブラウザの User-Agent を詐称する。これは個人アーカイブ用途に合わせた意図的な選択だが、裏を返せば**招かれていないサイトに負荷をかけうる**ということ。

以下を守ること:

- **個人のアーカイブ・オフライン閲覧**目的で使う。大規模スクレイピングや他者コンテンツの再公開には使わない。
- 該当する場合は**対象サイトの利用規約と `robots.txt` を尊重**する。
- 接続レートは低めに保つ（デフォルトは 2/秒）。小規模サイトや非商用サイトを叩きすぎない。
- 自動ミラーを禁止しているサイトには使わない。

使い方の責任は利用者にある。

## 必要なもの

| 依存 | 用途 | インストール |
|---|---|---|
| `httrack` | ダウンロードエンジン（必須） | `brew install httrack` / `apt install httrack` / `dnf install httrack` / `pacman -S httrack` |
| `whiptail` | TUI 描画（必須） | macOS/Fedora: `brew install newt` / `dnf install newt` &nbsp; Debian/Ubuntu: `apt install whiptail` &nbsp; Arch: `pacman -S newt` |

`bash` 4 以上を想定。macOS の標準 `bash` は古い（3.2）ため、問題が出たら Homebrew で新しい bash を入れる（`brew install bash`）。

スクリプトは起動時に両方の依存をチェックし、検出したパッケージマネージャ経由でのインストールを提案する。

## インストール

```bash
git clone https://github.com/tkdesign-jp/universal-mirror.git
cd universal-mirror
chmod +x mirror.sh
./mirror.sh
```

初回実行時、スクリプトは自身のディレクトリに編集可能なフィルタファイルを2つ生成する:

- `exclusions.txt` — デフォルト除外パターン（SNS / トラッキング / アナリティクス / 検索）。
- `wayback_extras.txt` — Wayback モード専用の追加除外。

---

## 使い方

実行してプロンプトに従うだけ:

```bash
./mirror.sh
```

対話フロー（7ステップ）:

1. **URL** — 通常 URL または完全な Wayback URL を貼る。Wayback URL は自動判定される。
2. **Wayback モード** — 通常 URL の場合、ライブミラーとして扱うか、Wayback Machine 経由にするかを選択。
3. **日付パターン**（Wayback のみ）— `2011`（年全体・推奨）、`201105`（月）、`20110510`（特定日）。
4. **ドメイン限定** — クロール対象ドメインを限定（URL から自動検出）。
5. **出力ディレクトリ** — ミラーの保存先。既存の場合は、上書き / 再開（`--update`）/ キャンセルを選べる。
6. **除外ファイル** — デフォルト、スクリプトディレクトリ内の任意の `exclusions_*.txt`、またはカスタムパスを選択。
7. **性能設定** — 再帰深さと秒あたり接続数。
8. **実行モード** — フォアグラウンド（進捗表示）かバックグラウンド（`nohup`、監視コマンドを表示）。

どの画面でも **ESC** または **Cancel** でクリーンに中断できる。

### 中断したミラーの再開

既存の出力ディレクトリを指定し、**再開** を選ぶ。スクリプトはフルコマンドを再構築して `--update` を付与するので、HTTrack はキャッシュを再利用しつつ、今回変更した深さ / フィルタ / 除外も反映する。

---

## 除外パターンのカスタマイズ

`exclusions.txt` は HTTrack のフィルタ構文を使う。1行1パターン、`#` でコメント:

```
# ドメイン全体を除外
-*example-tracker.com*

# パスを除外
-*/ads/*
```

名前付きの除外セットを複数持ちたい場合は、`exclusions_<名前>.txt` という名前のファイルをスクリプトディレクトリに置く。ステップ6で選択肢として表示される。

---

## なぜ HTTrack なのか（wget ではなく）

Wayback ミラーや、リンク再マッピング付きの再帰オフライン閲覧については、非エキスパートにとって HTTrack の方が `wget --mirror` より手間がかからない傾向がある。`wget` はより軽量でスクリプタブルだが、より多くのフラグと手動のリンク処理を要する。このラッパーは「ガイド付き・オフライン閲覧」のユースケースのために、意図的に HTTrack を選んでいる。コマンドラインに住んでいて最大限の制御が欲しいなら、素の `wget` の方が合うかもしれない。それはそれで構わない。
